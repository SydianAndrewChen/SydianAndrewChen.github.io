## 模板



### 函数模板

```c++
template<typename T>
bool compare(const T & v1, const T & v2){
	return v1 < v2;
}
```

- 模板定义以关键字template开始，后跟一个模板参数列表，由一个逗号分隔的一个或多个模板参数的列表
- 在模板定义中，模板参数列表不能为空
- 在使用模板时，我们指定模板始参，将其绑定到模板参数上



#### 实例化函数模板

- 编译器会用推断出的函数模板那参数来为我们**实例化**一个特定版本的函数。
- 当编译器实例化一个模板时，它使用实际的模板实参代替对应的模板参数来创建出模板的一个新“实例”

```c++
// compare定义见上文
cout << compare(1,2) << endl; // 这里实例化出了 bool compare(const int &, const int &)

cout << compare("Hello", "Hi") << endl; // 这里实例化除了bool compare(const string &, const string &)
```

- 这些编译器生成的版本通常被称为模板的**实例**。

#### 模板类型参数

- 类型参数前必须加上关键字`class`或`typename`
  - 这两个关键字的含义相同，可以互换使用，一个模板参数列表里可以同时使用这两个关键字。

#### 非类型模板参数

- 一个非类型参数表示一个值而非一个类型

- 当一个模板被实例化时，非类型参数被一个用户提供的或编译器推断的值所代替。

  - 这些值必须是**常量表达式**，从而允许编译器在编译时就将模板实例化。

  ```c++
  template<unsigned N, unsigned M>
  bool compare(const char (&p1)[N], const char (&p2)[M]){
  	return strcmp(p1, p2);
  }
  
  int main(){
      compare("hi", "hello");
      // 编译器首先在字符串常量后面加上一个终结符
      // 然后实例化一个实例：
      // bool compare(const char (&p1)[3], const char (&p2)[4]);
  }
  ```

  - 非类型参数可以是整型或者一个指向对象或函数类型的指针或者左值引用
    - 绑定到非类型整型参数的实参必须是一个常量表达式
    - 绑定到非类型指针或左值引用的实参必须拥有静态的生命周期。因此我们不能用一个局部变量或者动态对象作为指针或引用非类型模板参数的实参。
    - 指针类型可以用nullptr或者0值来实例化

#### 模板编译

当编译器遇到一个模板定义时，它并不生成代码。只有当我们实例化出模板的一个特定版本时，编译器才会生成代码。

- 在通常情况下，我们调用一个函数时，只需要在当前文件中有这个函数的声明即可，编译器会为我们寻找当前函数的定义。
- 但因为模板编译的特殊性，编译器并不生成代码保留模板的定义，因此我们必须在模板的头文件中即包括声明也包括定义。



#### 大多数模板编译错误在实例化期间报告

- 三个阶段
  - 模板定义：编译器可以帮忙检查一些简单的语法错误
  - 模板使用：编译器可以帮忙检查一些参数数目类型错误
  - 模板实例化：编译器才能发现大多数类型相关的错误



### 类模板

- 与函数模板的不同之处：编译器不能为类模板推断模板参数类型

  因此我们在使用类模板的时候必须在模板名后的尖括号内提供额外信息

  ```c++
  compare(2,3); // 函数模板，可以不加尖括号
  vector<int> vec; // 类模板，必须加尖括号
  ```

#### 实例化类模板

我们提供的信息是**显式模板实参列表**。它们被绑定到模板参数，编译器使用这些模板实参来实例化出特定的类。

- 一个类模板的每个实例都形成一个独立的类，`vector<int>` 和`vector<string>`与其他任何`vector`类型都没有关联，也不会对其他任何`vector`类型的成员有特殊访问权限

#### 类模板的成员函数

类模板的成员函数本身是一个普通函数。但是，类模板的每个实例都有其自己版本的成员函数。因此，类模板的成员函数具有和模板相同的模板参数。因此定义在类模板之外的成员函数就必须以关键字template开始，后接模板参数列表。

```c++
template <typename T> myClass{
	void check() const;
    
};

template <typename T>
void myClass<T>::check() const
{
	// implementations
}
```

默认情况下，一个类模板成员函数**只有当程序用到它的时候才进行实例化**。

如果一个成员函数没有被使用，就不会被实例化。

这一特性使得即使某些类型完全不符合模板操作的要求，我们仍然能用该类型实例化类。



#### 在类模板代码内部简化模板类名的使用

当我们使用一个类模板类型时，必须提供模板实参，但这其实有一个例外，就是当我们在类模板的作用域内部时

```c++
template<typename T>
class myClass{
    // 可以省略模板参数
	myClass & operator=(const myClass & rhs);
    myClass * operator&(){
        // implementaion
    }
};

template <typename T> // 不能省略模板参数
myClass<T> & myClass<T>::operator=(const myClass<T> & rhs){
    // 可以省略模板参数
    myClass tmp = *this;
    // Implementations
}
```



### 模板参数

模板参数在模板作用域内也存在隐藏和不允许重定义问题。

模板声明必须包含模板参数，模板声明和模板定义中的参数名不一定相同（但类型，数量，顺序一定要相同）

#### 使用类的类型成员

```c++
template<typename T>
...
/*
T::size_type * p;
编译器会感到困惑：这里究竟是一个指针声明还是一次乘法运算呢？因此我们需要显式告知编译器这个名字是一个类型还是一个变量
改为：
*/
typename T::size_type * p;
...
```

- 想要显式告知编译器一个名字表示类型时，必须使用typename而不能使用class

#### 默认模板实参

- 与函数默认实参一样，对于一个模板参数，只有当它的右侧所有参数都有默认实参时，它才可以有默认实参



### 控制实例化

当模板被使用时才会进行实例化。这一特性意味着，相同的实例可能出现在了多个对象文件中。当两个或多个独立编译的源文件使用了相同的模板，并提供了相同的模板参数时，每个文件中就会有该模板的一个实例。

这样的开销可能会非常严重。在新标准中，我们可以通过**显式实例化**来避免这种开销。
